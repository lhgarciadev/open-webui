# Fase 2.1 - Ejecutar Configuración de ZeroGPU

## Objetivo
Habilitar ZeroGPU (GPU H200 gratuita) en el Space para acelerar la inferencia.

## Contexto
- ZeroGPU proporciona GPU NVIDIA H200 (~70GB VRAM) bajo demanda
- Es gratuito con cuota diaria
- PRO ($9/mes) da 8x más cuota y prioridad

## Prompt de Ejecución

```
Habilita ZeroGPU en el Space de Ollama:

1. ACCEDER A SETTINGS DEL SPACE:

   # Abrir en navegador (reemplaza TU_USUARIO)
   # https://huggingface.co/spaces/TU_USUARIO/cognitia-ollama

   # Click en "Settings" (icono de engranaje)

2. CAMBIAR HARDWARE A ZEROGPU:

   a) En la sección "Space Hardware"
   b) Click en el dropdown de hardware
   c) Seleccionar: "ZeroGPU" (o "zero-a10g")
   d) Confirmar el cambio

   # El Space se reiniciará automáticamente

3. ESPERAR REINICIO:

   # El Space pasará por estos estados:
   # 1. Paused
   # 2. Building
   # 3. Running

   # Tiempo estimado: 2-5 minutos

4. VERIFICAR HARDWARE EN UI:

   # En la página del Space, debe mostrar:
   # Hardware: ZeroGPU
   # (o similar indicando GPU)

5. VERIFICAR EN LOGS:

   # Click en "Logs"
   # Buscar líneas que indiquen GPU:
   # - "CUDA available"
   # - "GPU detected"
   # - O simplemente que inicia sin errores

6. ALTERNATIVA VÍA README.md:

   # Si el UI no permite cambiar, editar README.md:

   cd ~/cognitia-ollama-space

   # Editar README.md, cambiar suggested_hardware:
   # suggested_hardware: zero-a10g

   # Commit y push
   git add README.md
   git commit -m "feat: Enable ZeroGPU hardware"
   git push

7. VERIFICAR CUOTA:

   # Ver uso de cuota en:
   # https://huggingface.co/settings/billing

   # Sección "ZeroGPU Usage"
   # Muestra cuota diaria disponible

Entrega:
- Screenshot del Space con ZeroGPU habilitado
- Logs mostrando inicio exitoso
- Confirmación de cuota disponible
```

## Opciones de Hardware ZeroGPU

| Opción | GPU | VRAM | Cuota |
|--------|-----|------|-------|
| `zero-a10g` | A10G (recomendado) | 24GB | Eficiente |
| `zero-a100` | A100 | 80GB | Alto consumo |

**Recomendación**: Usar `zero-a10g` para balance óptimo.

## Troubleshooting

### Opción ZeroGPU no aparece
```bash
# Puede requerir verificación de cuenta
# Ir a: https://huggingface.co/settings/account
# Verificar email y/o agregar método de pago

# Alternativa: usar README.md
# suggested_hardware: zero-a10g
```

### Space no reinicia
```bash
# Forzar reinicio manual
# Settings → "Factory reboot"

# O recrear el Space
huggingface-cli repo delete cognitia-ollama --type space -y
# Y repetir Fase 1
```

### Cuota agotada
```bash
# Opciones:
# 1. Esperar al día siguiente (reset diario)
# 2. Upgrade a PRO ($9/mes) para 8x cuota
# 3. Usar CPU temporalmente (más lento)
```

## Notas sobre ZeroGPU

- **Cuota diaria**: Se resetea cada 24 horas
- **PRO ($9/mes)**: 8x más cuota + prioridad en cola
- **Sleep mode**: Normal, el Space duerme cuando no se usa
- **Cold start**: ~10-30 segundos al despertar
