# Etapa 4.3 - Validar Temas con Backgrounds Apropiados

## Objetivo
Verificar que cada tema (dark, light, corporate) tiene backgrounds coherentes y texto legible.

## Prompt de Validacion

```
Valida la implementacion de temas en Reveal.js:

1. GENERAR PRESENTACIONES DE PRUEBA:

   Para cada tema, generar una presentacion:

   # Tema Dark
   result_dark = await generate_presentation(
       title="Test Tema Dark",
       slides=[
           {"type": "title", "title": "Tema Oscuro", "subtitle": "Fondo #0f172a"},
           {"type": "content", "title": "Contenido", "bullets": [
               "Texto primario visible",
               "Texto secundario legible",
               "Accent color destaca"
           ]},
           {"type": "stats", "title": "Metricas", "stats": [
               {"value": "100%", "label": "Contraste"}
           ]},
           {"type": "closing", "title": "Gracias"}
       ],
       theme="dark"
   )

   # Tema Light
   result_light = await generate_presentation(
       title="Test Tema Light",
       slides=[...mismo contenido...],
       theme="light"
   )

   # Tema Corporate
   result_corporate = await generate_presentation(
       title="Test Tema Corporate",
       slides=[...mismo contenido...],
       theme="corporate"
   )

2. VERIFICAR BACKGROUNDS:

   | Tema | Background Esperado | Background Actual | PASS/FAIL |
   |------|---------------------|-------------------|-----------|
   | dark | #0f172a | | |
   | light | #ffffff | | |
   | corporate | #1e40af | | |

3. VERIFICAR CONTRASTE (WCAG AA):

   Usar herramienta de contraste o inspeccion visual:

   | Tema | Elemento | Color Texto | Fondo | Ratio | PASS/FAIL |
   |------|----------|-------------|-------|-------|-----------|
   | dark | heading | #ffffff | #0f172a | | |
   | dark | body | #f8fafc | #0f172a | | |
   | light | heading | #1e293b | #ffffff | | |
   | light | body | #0f172a | #ffffff | | |
   | corporate | heading | #ffffff | #1e40af | | |
   | corporate | body | #ffffff | #1e40af | | |

   Objetivo: Ratio >= 4.5:1 para texto normal

4. VERIFICAR ELEMENTOS VISUALES:

   Para cada tema verificar:
   [ ] Stat cards tienen fondo visible
   [ ] Footer es legible
   [ ] Progress bar visible
   [ ] Gradientes armonizan con fondo
   [ ] Links tienen color distintivo

5. VERIFICAR FALLBACK:

   # Tema invalido debe usar dark
   result_invalid = await generate_presentation(
       title="Test Tema Invalido",
       slides=[{"type": "title", "title": "Test"}],
       theme="inexistente"
   )
   # Verificar que usa configuracion dark

6. ABRIR EN NAVEGADOR:

   - Abrir cada HTML generado
   - Verificar rendering en Chrome, Firefox, Safari
   - Verificar en mobile (responsive)

Entrega:
- 3 archivos HTML de prueba
- Tabla de contraste completada
- Screenshots de cada tema
- PASS/FAIL global
```

## Criterios de Exito

| Criterio | Descripcion | PASS/FAIL |
|----------|-------------|-----------|
| Background dark | Fondo #0f172a aplicado | |
| Background light | Fondo #ffffff aplicado | |
| Background corporate | Fondo #1e40af aplicado | |
| Contraste dark | Ratio >= 4.5:1 | |
| Contraste light | Ratio >= 4.5:1 | |
| Contraste corporate | Ratio >= 4.5:1 | |
| Stat cards | Visibles en todos los temas | |
| Footer | Legible en todos los temas | |
| Fallback | Tema invalido usa dark | |
