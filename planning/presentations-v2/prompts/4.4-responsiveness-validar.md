# Etapa 4.4 - Validar Responsividad del Chat

## Objetivo
Verificar que el sidebar no se sobrepone al contenido del chat en ninguna resolucion.

## Prompt de Validacion

```
Valida la responsividad del chat y sidebar:

1. PREPARAR ENTORNO DE PRUEBA:

   - Abrir Open WebUI en navegador
   - Tener DevTools abierto (F12)
   - Activar Device Toolbar (Ctrl+Shift+M)

2. TEST EN MOBILE (375px - iPhone SE):

   a) Estado inicial:
      [ ] Sidebar oculto por defecto
      [ ] Contenido del chat ocupa 100% width
      [ ] Hamburger menu visible

   b) Abrir sidebar:
      [ ] Sidebar aparece como drawer (desde izquierda)
      [ ] Backdrop oscuro visible detras
      [ ] Contenido del chat NO se mueve
      [ ] Contenido del chat queda detras del backdrop

   c) Cerrar sidebar:
      [ ] Click en backdrop cierra sidebar
      [ ] Sidebar se desliza hacia izquierda
      [ ] Contenido vuelve a ser interactivo

3. TEST EN TABLET (768px - iPad):

   a) Portrait:
      [ ] Mismo comportamiento que mobile
      [ ] Drawer + backdrop

   b) Landscape (1024px):
      [ ] Sidebar fijo a la izquierda
      [ ] Sin backdrop
      [ ] Contenido tiene margin-left

4. TEST EN DESKTOP (1280px+):

   [ ] Sidebar siempre visible
   [ ] Contenido tiene margin-left apropiado
   [ ] Sin superposicion
   [ ] Toggle sidebar funciona (opcional)

5. TEST DE INTERACCION:

   a) Con sidebar abierto en mobile:
      [ ] NO se puede hacer scroll en contenido
      [ ] NO se puede hacer click en chat
      [ ] Solo sidebar es interactivo

   b) Links en sidebar:
      [ ] Click en chat de la lista cierra sidebar
      [ ] Navegacion funciona correctamente

6. TEST DE Z-INDEX:

   Orden de capas (verificar con DevTools):
   1. Contenido principal: z-index ~1
   2. Backdrop: z-index ~40
   3. Sidebar drawer: z-index ~50

   [ ] Sidebar esta encima del backdrop
   [ ] Backdrop esta encima del contenido
   [ ] Modales (si hay) estan encima de todo

7. TEST DE TRANSICIONES:

   [ ] Sidebar tiene animacion suave (300ms aprox)
   [ ] Backdrop tiene fade suave
   [ ] Sin saltos o glitches visuales
   [ ] Sin parpadeos

8. TEST DE ROTACION:

   En dispositivo real o simulador:
   [ ] Rotar de portrait a landscape
   [ ] Sidebar se adapta correctamente
   [ ] Sin errores de layout

9. CAPTURAS DE PANTALLA:

   Tomar screenshot de:
   - Mobile sidebar cerrado
   - Mobile sidebar abierto
   - Tablet landscape
   - Desktop

Entrega:
- Checklist completado
- Screenshots de cada resolucion
- Lista de issues encontrados (si hay)
- PASS/FAIL global
```

## Breakpoints Esperados

```css
/* Mobile first */
.sidebar {
  /* Default: drawer overlay */
  position: fixed;
  transform: translateX(-100%);
}

.sidebar.open {
  transform: translateX(0);
}

/* Desktop: sidebar fijo */
@media (min-width: 1024px) {
  .sidebar {
    transform: translateX(0);
    position: relative; /* o fixed con margin en content */
  }

  .main-content {
    margin-left: 280px; /* width del sidebar */
  }
}
```
