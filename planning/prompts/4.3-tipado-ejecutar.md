# Etapa 4.3 - Ejecutar Saneamiento de Tipado

## Prompt
Reduce la deuda de errores preexistentes de `npm run check` en una etapa controlada:

1. Separar ruido legacy de JS estricto:
   - ajustar estrategia de `checkJs` para no bloquear el pipeline global.
2. Corregir errores de alto impacto en rutas/core UI (prioridad en settings/auth/layout).
3. Dejar plan incremental para migracion de modulos legacy (`.js` -> `.ts`) por lotes.
4. Mantener compatibilidad funcional (sin romper flujos de login/chat/settings).

Entrega:
- Cambios concretos en configuracion y/o codigo para reducir errores.
- Conteo antes/despues de errores de `npm run check`.
- Backlog priorizado de archivos pendientes (Top 10 por impacto).
