# Etapa 5.2 - Validar Docker

## Prompt

Valida que los builds y despliegues Docker funcionen en modo standard y white-label:

1. `docker build -t cognitia .` y `docker build -f Dockerfile.whitelabel -t cognitia .`
2. `docker-compose up` y `docker-compose -f docker-compose.whitelabel.yaml up` (smoke test)
3. Verificar healthcheck/logs de inicio y que el frontend cargue.
4. Confirmar que no hay referencias a "Open WebUI" en UI/branding durante el build.

Entrega:

- PASS/FAIL por build y compose.
- Errores encontrados + fix sugerido.
- Riesgos pendientes.
